## Идея проекта

Новостной сайт **iPhoneFeed.news** (ранее AppleDaily.news) — агрегатор новостей из мира Apple и технологий.

**Как работает:**
1. Python-скрипт парсит RSS-фиды топовых англоязычных источников (MacRumors, 9to5Mac, AppleInsider, The Verge, Ars Technica)
2. GPT API автоматически переводит и делает рерайт на русский язык
3. Контент автоматически публикуется на WordPress через REST API
4. Периодичность: каждые 2 часа (cron)

**Монетизация:** Яндекс РСЯ + Google AdSense на старте. Нативная реклама — когда вырастет аудитория (50k+ уников/мес).

**Целевая аудитория:** Русскоязычные пользователи Apple — в России, Израиле и других странах.

## Следующие шаги

### Шаг 1 — Создать WordPress Application Password ⏳
1. Зайти в WP Admin → Пользователи → Профиль
2. Прокрутить до раздела "Application Passwords"
3. Ввести имя "autoposter" → нажать "Add New Application Password"
4. Скопировать сгенерированный пароль (вида `xxxx xxxx xxxx xxxx`)
5. Сохранить в файл `.env` (скопируй `.env.example` → `.env`)

### Шаг 2 — Загрузить код на GitHub ⏳
1. Создать репозиторий на github.com (приватный!)
2. `git init && git add . && git commit -m "init"`
3. `git remote add origin https://github.com/ТВО_ИМЯ/appledaily && git push`

### Шаг 3 — Добавить GitHub Secrets ⏳
В репозитории → Settings → Secrets → Actions → New secret:
- `WP_URL` = `https://iphonefeed.news`
- `WP_USER` = `admin`
- `WP_PASSWORD` = Application Password из шага 1
- `OPENAI_API_KEY` = ключ OpenAI

### Шаг 4 — Тестовый запуск ⏳
- В GitHub → Actions → "AppleDaily Autoposter" → Run workflow
- Проверить что статьи появились на сайте

### Шаг 5 — Подключение рекламы ⏳
- Зарегистрироваться в Яндекс РСЯ
- Зарегистрироваться в Google AdSense
- Вставить коды в шаблон на место рекламных слотов

### Шаг 6 — Google News ⏳
- Подать заявку на включение в Google News (после ~10 публикаций)
- Это даст органический трафик без SEO-усилий

## Юридические моменты

- Использовать RSS легально — это публичные фиды
- Делать рерайт (не просто перевод) — это важно для защиты от DMCA
- Указывать источник в каждой статье ("via MacRumors") — хорошая практика
- В футере написать: "Переводы с MacRumors, 9to5Mac, The Verge и других"
